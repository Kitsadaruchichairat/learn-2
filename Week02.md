> # Semantic Versioning (Semver) คือ?
> Semver เป็นมาตรฐานในการตั้งหมายเลขเวอร์ชั่นของServer เพื่อให้ผู้พัฒนาและผู้ใช้งานรู้ว่า การอัปเดตครั้งนี้เปลี่ยนแปลงมากน้อยแค่ไหน จากตัวเลขเวอร์ชัน โดยไม่ต้องอ่านรายละเอียดทั้งหมด

> โดยโครงสร้าง ของ Semantic Versioning ประกอบด้วย 
```
MAJOR.MINOR.PATCH
```

> MAJOR หมายถึง มีการเปลี่ยนแปลงใหญ่ ที่อาจทําให้ของเดิมใช้งานไม่ได้

> เช่น 


- เปลี่ยนโครงสร้างระบบครั้งใหญ่
- เปลี่ยนวิธีใช้ API
- ลบฟังค์ชันเก่า
- เปลี่ยนรูปแบบข้อมูล
- ทําให้โค้ดเดิมต้องมีการแก้ไข 

> ของเดิมอาจใช้ไม่ได้ เพิ่ม MAJOR

> MINOR หมายถึง เพิ่มความสามารถใหม่ แต่ยังไม่ทําลายของเดิม

> เช่น


- เพิ่มฟีเจอร์ใหม่
- เพิ่มเครื่องมือใหม่
- เพิ่มความสามาถเพิ่มเติม

> ของเก่ายังใช้ได้ ให้ใช้เป็น MINOR

> PATCH หมายถึง แก้ไขเล็กน้อย

> เช่น


- แก้บั๊ก
- ปรับความปลอดภัย
- แก้คําผิด
- ปรับประสิทธิภาพ

> แก้ไขบั๊ก/ปรับปรุง ให้ใช้เป็น PATCH

> ## กฏของการใช้ SemVer
1. การ PATCH ให้เพิ่มทีละ 1


เช่น
```
1.0.0  ไป 1.0.1 ไป 1.0.2 
```

2. เมื่อเพิ่ม MINOR ต้องรีเซ็ต PATCH 

เช่น
```
1.4.8 ไป 1.5.0 
```

3. เมื่อเพิ่ม MAJOR ต้องรีเซ็ต MINOR และ PATCH

เช่น
```
2.9.5 ไป 3.0.0
```

> เวอร์ชันพิเศษ
บางครั้งก็มีตัวอักษรต่อท้าย 

เช่น
```
1.5.0-beta
```

> หมายถึงเวอร์ชั่นทดลอง โดยที่ 
- alpha = ทดลองช่วงแรก
- beta = ทดลองการใช้งาน
- rc (Release Candidate) = ใกล้ช่วงปล่อยจริง


># Web Server คือ?
> คือที่เก็บซอฟต์แวร์และไฟล์ต่างๆ ที่เป็นองค์ประกอบของเว็บไซต์( HTML, CSS,Image เป็นต้น) โดยเว็บเซิร์ฟเวอร์ทําหน้าที่ส่งหน้าเว็บไปยังผู้ใช้(Client) ตามคําร้องขอ (request) ของผู้ใช้ผ่านเว็บเบราว์เซอร์ เช่นGoogle Chrome แล้วส่งข้อมูลกลับไป (Response)

>### Web Server ยอดนิยม

> 1. Nginx
เป็น Web Server อันดับ1 ที่ได้รับความนิยมสูงสุด ด้วยความเร็ว ความยืดหยุ่น และประสิทธิภาพสูง โดยเฉพาะการปรับแต่ง Nginx ร่วมกับ modules และเทคโนโลยีใหม่ๆ และ ทําให้สามารถรองรับ กับเว็บไซต์ hight traffic ได้ดียิ่งขึ้น

> 2. Apache 
Apache HTTP Server เป็นเว็บเซิร์ฟเวอร์ที่ได้รับความนิยมโดย ยังคงมีด้วยความเสถียร ความยืดหยุ่น และมีความปลอดภัย และ มีจุดแข็งที่มีCommunity ขนาดใหญ่และมีเสถียรภาพ

> 3. Microsoft IIS
มีฐานผู้ใช้ที่แข็งแกร่ง โดยเฉพาะในองค์กรที่ใช้ Microsoft เทคโนโลยีเป็นหลัก โดยมีความสามารถในการผสานการทํางานกับระบบนิเวศของ Windows Server และ .NET

3. จัดการไฟล์เว็บไซต์

- HTML

- JavaScript
- image
- video

 ```
 website/
│
├── index.html

```
4. เชื่อมต่อกับฐานช้อมูล เว็บไซต์ที่มีข้อมูลเปลี่ยนเปลง ตัว Web Server จะติดต่อกับตัว Database
```
ผู้ใช้
 ↓
Web Server
 ↓
Database
 ↓

```

> มี
- CPU

โปรแกรมที่จัดการเว็บไซต์

  - ใช้กับระบบ  Windows Server

> # HTTP Standard คือ 
> มาตรฐาน ที่กําหนด วิธี ที่บอกว่า Browser ต้องส่งคำขออย่างไร และ Server ต้องตอบกลับอย่างไร

>#### HTTP ย่อมาจาก Hypertext Transfer Protocol    
เป็นมาตรฐานการสื่อสารบนเว็บ ใข้ส่งข้อมูล ระหว่าง
```
Client (Browser)
        ↓
      HTTP
        ↓
Server
```

> ### ข้อกําหนดของ HTTP Standard

> ### 1.  Request
    > Request คือ ข้อมูลที่ Client (Browser) ส่งไปหา Server เพื่อขอให้ทำงานบางอย่าง เช่น
```HTTP
GET /products HTTP/1.1
Host: kit.com
```
>- GET = วิธีขอข้อมูล (HTTP Method)
>- /products = ไฟล์ที่ต้องการ
>- HTTP/1.1 = เวอร์ชันของ HTTP
>- Host → ชื่อเว็บไซต์

> ### Request มีส่วนประกอบหลัก ได้แก่
    >#### 1. Request Line
     

>#### ประกอบด้วย

```
Method + Path + HTTP Version
```
>#### เช่น

```http
GET /index.html HTTP/1.1
```

> #### 2. Header บอก ข้อมูลเพิ่มเติมของ Request
>#### เช่น
```http
User-Agent: Chrome
Content-Type: application/json
```
ใช้บอก
>- Browser ที่ใช้งาน
>- ประเภทของข้อมูล
>- ข้อมูลการเชื่อมต่อ

> #### 3. Body คือข้อมูลที่ส่งกลับ
>#### เช่น


```HTML
<h1>Hello World</h1>
```

>   ###  2.  HTTP Methods
HTTP กําหนดวิธีการทํางาน



| Method | ใช้ทำอะไร | ตัวอย่าง |
| ---| --- | --- |
| **GET** | ขอข้อมูล | ดูสินค้าเปิดหน้าเว็บ |
| **POST** | ส่งข้อมูล / สร้างข้อมูลใหม่ | สมัครสมาชิก, เพิ่มสินค้า |
| **PUT** | แก้ไขข้อมูลทั้งหมด | แก้ไขข้อมูลผู้ใช้ทั้งชุด |
| **PATCH** | แก้ไขข้อมูลบางส่วน | เปลี่ยนแค่ชื่อผู้ใช้ |
| **DELETE** | ลบข้อมูล | ลบสินค้า, ลบบัญชี |
| **HEAD**  | ขอข้อมูลส่วนหัว ไม่เอาเนื้อหา  | ตรวจสอบว่าไฟล์มีอยู่ไหม |
| **OPTIONS** | ตรวจสอบว่ารองรับคำสั่งอะไรบ้าง | ดูว่า API ใช้ GET/POST ได้ไหม |
> #### ตัวอย่าง

```
GET /products
(ขอสินค้า)
```

```
POST /products
(เพิ่มสินค้า)
```

> ### 3. Status Code
> HTTP Standard กำหนดรหัสตอบกลับ
```
200 ok (ทํางานสําเร็จ)
``` 

```
301 Moved Permanently (ย้ายหน้าเว็บ)
```

```
400 Bad Request
(ส่งข้อมูลผิด)

404 Not Found
(ไม่พบหน้าเว็บ)
```

```
500 Internal Server Error
(Server มีปัญหา)
```

> ### HTTP Standard เกี่ยวข้องกับ HTTPS อย่างไร?
> HTTP Standard คือกติกาการสื่อสาร 
> ส่วน HTTPS คือ HTTP ที่เพิ่มความปลอดภัย


```
HTTP Standard
      +
SSL/TLS
      ↓
HTTPS
```

> ### ส่วนประกอบของ URL
```
URL (Uniform Resource Locator) คือ ที่อยู่ของทรัพยากรบนเว็บไซต์
```

> #### ตัวอย่าง 
```
https://www.kitsada.com/products?id=10
```
> #### แบ่งได้เป็น
```text
https://www.kitsada.com/products?id=10
│       │                 │        │
│       │                 │        └── Query
│       │                 └─────────── Path
│       └───────────────────────────── Domain
└───────────────────────────────────── Protocol
```

> #### รายละเอียด
| ส่วนประกอบ | ความหมาย |
|---|---|
| `https://` | Protocol ที่ใช้สื่อสาร |
| `www.kitsada.com` | Domain / ชื่อเว็บไซต์ |
| `/products` | Path ตำแหน่งข้อมูล |
| `?id=10` | Query Parameter |

> ### ความสัมพันธ์ Request กับ URL

เวลาเปิดเว็บ เช่น
```
https://kit.com/products
```
Browser จะสร้าง Request

```http
GET /products HTTP/1.1
Host: kit.com

(ส่งไปหา Server)
```
Server ตอบกลับ
```http
HTTP/1.1 200 OK
```



> # SSL คืออะไร?

>Secure Sockets Layer(SSL) คือ โปรโตคอลความปลอดภัยที่ใช้เข้ารหัสข้อมูล ระหว่าง Client กับ Server 

หรือง่ายๆคือ 
```
SSL คือระบบที่สร้าง “ช่องทางสื่อสารที่ปลอดภัย” ระหว่างผู้ใช้กับเว็บไซต์
```

> #### ตัวอย่าง
ก่อนใช้ SSL 
```
Browser
   |
   |  username = admin
   |  password = 1234
   ↓
Server

(ข้อมูลอาจถูกดักอ่าน)
```
เมื่อใช้ SSL
```
Browser
   |
   |  #@8xK91$!a
   ↓
Server

(ข้อมูลถูกเข้ารหัส คนอื่นที่ดักได้จะอ่านไม่ออก)
```

> ### SSL เกี่ยวข้องกับ HTTPS อย่างไร?
HTTPS เกิดจาก
```
HTTP + SSL/TLS = HTTPS
```
โดย

>- HTTP = มาตรฐานการการส่งข้อมูล
>- SSL/TLS = ระบบรักษาความปลอดภัย
>- HTTPS = HTTP ที่มีการป้องกัน 

> ### หน้าที่หลักของ SSL มี 3 อย่าง


> 1. Encryption(การเข้ารหัส)
ทําให้ข้อมูลที่ส่งผ่านอินเทอร์เน็ตอ่านไม่ได้
>#### ตัวอย่าง
ข้อมูลจริง
```
password=123456
```
หลังเข้ารหัส
```
x8#kP29@zL
```
> 2. Authentication (การยืนยันตัวตน)
ตรวจสอบว่าเว็บไซต์เป็นของจริง

>#### เช่น เข้า
```
https://bank.com
```
SSL Certification จะช่วยตรวจสอบว่า
> - เว็บไซต์นี้เป็นของธนาคารจริงไหม 
> - ไม่ใช่เว็บไซต์ปลอม




> 3. Data Integrity (ความถูกต้องของข้อมูล)

ป้องกันข้อมูลถูกแก้ไขระหว่างทาง

> #### ตัวอย่าง 

> Server ส่ง
```
ราคา = 100 บาท
```
> ถ้ามีคนพยายามแก้เป็น
```
ราคา = 1000 บาท
```
> ระบบจะตรวจพบว่าข้อมูลถูกเปลี่ยน

> SSL Certificate คือ ใบรับรองดิจิทัล ที่ใช้ยืนยันตัวตนของเว็บไซต์

> ใน Certificate จะมีข้อมูล เช่น:

>- ชื่อเว็บไซต์
>- เจ้าของเว็บไซต์
>- Public Key
>- ผู้ออกใบรับรอง

>เมื่อ Browser เข้าเว็บ จะตรวจสอบ Certificate ก่อน

>### SSL ทำงานอย่างไร

> #### 1. Browser ขอเชื่อมต่อ
```
Client → Server
```

> #### 2. Server ส่ง Certificate
Server ส่ง ให้ Browser
```
SSL Certificate + Public Key
```
> #### 3. Browser ตรวจสอบ Certificate
> - Certificate ถูกต้องไหม
> - ออกโดย CA ที่เชื่อถือไหม
> - ชื่อเว็บตรงไหม

> ##### CA หรือ Certificate Authority  เป็นหน่วยงานที่ออก SSL Certificate

เช่น
 DigiCert, GlobalSign, Let's Encrypt
  
  ทําหน้าที่ 

> - ตรวจสอบเว็บไซต์
> - ออกใบรับรอง
> - รับรองว่าเว็บไซต์นั้นเป็นของจริง


> #### 4. สร้างกุญแจเข้ารหัส

> ทั้งสองฝั่งตกลงวิธีเข้ารหัสร่วมกัน
> จากนั้นเริ่มส่งข้อมูลแบบปลอดภัย

> ### TLS คืออะไร 
> Transport Layer Security (TLS) คือ โปรโตคอลรักษาความปลอดภัยที่ใช้เข้ารหัสข้อมูลระหว่าง Client กับ Server ผ่านเครือข่ายอินเทอร์เน็ต
> เป็นเทคโนโลยีใหม่ที่พัฒนาต่อจากSSL ทําหน้าที่เหมือน SSL 

> #### SSL กับ TLS ต่างกันอย่างไร?

> SSL เป็นเทคโนโลยีรุ่นเก่า
> ปัจจุบันใช้ Transport Layer Security (TLS) แทน 

```
SSL 1.0
SSL 2.0
SSL 3.0
      ↓
TLS 1.0
TLS 1.2
TLS 1.3
```
> ปัจจุบันเว็บไซต์ส่วนใหญ่ใช้ TLS แต่คนทั่วไปยังเรียกว่า SSL

> #### ตัวอย่างการใช้งาน SSL
>- เว็บไซต์ธนาคาร
>- ร้านค้าออนไลน์
>- ระบบ Login
>- Email
>- API
>- ระบบชำระเงิน


> ### เพิ่มเติม 

>#### SSL 

>####  http status code ส่วนที่เจอบ่อยๆใน production

| Code | ความหมาย           | เจอบ่อย     |
| ---- | ------------------ | ----------- |
| 200 Created | สำเร็จ             | API GET     |
| 201  | สร้างสำเร็จ        | POST        |
| 204  | สำเร็จไม่มีข้อมูล  | DELETE      |
| 304  | ข้อมูลไม่เปลี่ยน ใช้ Cache เดิม  | Browser / CDN      |
| 400  | Request ผิด        | Validation  |
| 401  | ยังไม่ Login       | JWT         |
| 403  | ไม่มี Permission   | Role        |
| 404  | ไม่พบข้อมูล        | URL/ID      |
| 409  | ข้อมูลซ้ำ          | Register    |
| 500  | Server Error       | Bug         |


# SSL/TLS Handshake

>SSL/TLS Handshake คือกระบวนการที่ Client และ Server ตกลงวิธีสื่อสารแบบปลอดภัยก่อนส่งข้อมูลจริง

>เป้าหมาย:
- ยืนยันตัวตน Server
- แลกเปลี่ยน Key
- สร้างช่องทางเข้ารหัส

![HandShake](https://images.ctfassets.net/slt3lc6tev37/5aYOr5erfyNBq20X5djTco/3c859532c91f25d961b2884bf521c1eb/tls-ssl-handshake.png)
>Flow:
```
Client
 |
 | 1. Client Hello
 | - ส่ง TLS Version
 | - ส่ง Encryption ที่รองรับ
 |
 v

Server
 |
 | 2. Server Hello
 | - เลือก Algorithm
 | - ส่ง Certificate
 |
 v

Client
 |
 | 3. ตรวจสอบ Certificate
 | - ตรวจว่า Certificate ถูกต้อง
 | - ตรวจ CA
 |
 v

Client + Server
```
